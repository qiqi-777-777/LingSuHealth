# 问题修复总结

## 问题描述
1. 后端启动失败 - 端口 8081 被占用
2. 前端页面节气挂件不显示

## 问题原因

### 1. 后端启动失败
- **原因**：端口 8081 已被旧的 Java 进程（PID 25100）占用
- **错误信息**：`Web server failed to start. Port 8081 was already in use.`

### 2. 节气挂件不显示
- **原因**：后端未启动，前端无法获取节气数据
- API 端点：`/api/season/current`

## 解决方案

### 1. 停止占用端口的进程
```bash
# 查找占用端口的进程
netstat -ano | findstr :8081

# 停止进程
taskkill /F /PID 25100
```

### 2. 重新启动后端
```bash
mvn spring-boot:run
```
- ✅ 后端成功启动在端口 8081
- ✅ API 测试通过：`http://localhost:8081/api/season/current`
- ✅ 返回当前节气数据（立春）

### 3. 启动前端
```bash
cd ui
npm run dev
```
- ✅ 前端成功启动在端口 5174（5173 被占用，自动切换）
- ✅ 可访问：http://localhost:5174/
