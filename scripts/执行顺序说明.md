# 数据库脚本执行顺序

## ⚠️ 重要提示
必须按照以下顺序执行脚本，否则会出现外键约束错误！

## 执行步骤

### 第一步：备份数据库（必须！）
```bash
mysqldump -u root -p lingsu_health > backup_$(date +%Y%m%d_%H%M%S).sql
```

### 第二步：修改 users 表类型（前置条件）
```bash
mysql -u root -p lingsu_health < scripts/migrate_users_id_to_bigint.sql
```

**说明**：将 `users.id` 从 `INT` 改为 `BIGINT`，这是后续所有表的前置条件。

### 第三步：创建每日任务表
```bash
mysql -u root -p lingsu_health < scripts/create_daily_tasks_table.sql
```

**说明**：创建 `user_daily_tasks` 表，带外键约束。

### 第四步：健康闭环优化（可选）
```bash
mysql -u root -p lingsu_health < scripts/health_loop_optimization.sql
```

**说明**：创建健康计划、周报、提醒等表，完善健康管理闭环。

## 验证结果

执行以下 SQL 验证所有表是否创建成功：

```sql
-- 检查 users.id 类型
SELECT COLUMN_NAME, DATA_TYPE, COLUMN_TYPE 
FROM information_schema.COLUMNS 
WHERE TABLE_SCHEMA = 'lingsu_health' 
  AND TABLE_NAME = 'users' 
  AND COLUMN_NAME = 'id';

-- 检查所有新表
SELECT TABLE_NAME, TABLE_COMMENT 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'lingsu_health' 
  AND TABLE_NAME IN ('user_daily_tasks', 'health_plans', 'weekly_reports', 'scheduled_reminders');

-- 检查外键约束
SELECT 
    TABLE_NAME,
    CONSTRAINT_NAME,
    REFERENCED_TABLE_NAME,
    REFERENCED_COLUMN_NAME
FROM information_schema.KEY_COLUMN_USAGE
WHERE TABLE_SCHEMA = 'lingsu_health'
  AND REFERENCED_TABLE_NAME = 'users';
```

## 常见错误

### 错误 1：外键约束失败
```
ERROR: Foreign key constraint is incorrectly formed
```

**原因**：没有先执行 `migrate_users_id_to_bigint.sql`

**解决**：按顺序执行第二步

### 错误 2：表已存在
```
ERROR: Table 'xxx' already exists
```

**原因**：重复执行脚本

**解决**：脚本中已包含 `DROP TABLE IF EXISTS`，正常情况不会出现此错误

## 回滚方案

如果出现问题，使用备份恢复：

```bash
mysql -u root -p lingsu_health < backup_YYYYMMDD_HHMMSS.sql
```
