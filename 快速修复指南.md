# 数据库外键约束问题 - 快速修复指南

## 一键执行（推荐）

### Windows 系统
```cmd
REM 1. 备份数据库
mysqldump -u root -p lingsu_health > backup_%date:~0,4%%date:~5,2%%date:~8,2%.sql

REM 2. 执行迁移（按顺序）
mysql -u root -p lingsu_health < scripts\migrate_users_id_to_bigint.sql
mysql -u root -p lingsu_health < scripts\create_daily_tasks_table.sql

REM 3. 验证结果
mysql -u root -p lingsu_health < scripts\test_migration.sql
```

### Linux/Mac 系统
```bash
# 1. 备份数据库
mysqldump -u root -p lingsu_health > backup_$(date +%Y%m%d).sql

# 2. 执行迁移（按顺序）
mysql -u root -p lingsu_health < scripts/migrate_users_id_to_bigint.sql
mysql -u root -p lingsu_health < scripts/create_daily_tasks_table.sql

# 3. 验证结果
mysql -u root -p lingsu_health < scripts/test_migration.sql
```

## 问题原因
- `users.id` 是 `INT` 类型
- `user_daily_tasks.user_id` 是 `BIGINT` 类型
- 类型不匹配，无法创建外键

## 解决方案
将 `users.id` 改为 `BIGINT`，统一所有类型

## 修改的文件
✅ `src/main/java/com/lingsu/health/entity/DailyTask.java` - userId 改为 Long
✅ `scripts/create_daily_tasks_table.sql` - 添加外键约束
✅ `scripts/migrate_users_id_to_bigint.sql` - 数据库迁移脚本（新建）

## 执行后验证
1. 检查 users.id 类型是否为 BIGINT
2. 检查外键约束是否创建成功
3. 重启后端应用
4. 测试每日任务功能

## 如果出错
使用备份恢复：
```bash
mysql -u root -p lingsu_health < backup_YYYYMMDD.sql
```
