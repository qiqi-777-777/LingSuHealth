目标：知识库场景化 ：将知识库内容按“人群/症状/季节/目标”维度做场景入口，降低搜索成本。

总体思路

- 把“搜索”从唯一入口，变成“场景入口 + 搜索兜底”：用户先选人群/症状/季节/目标（任意 1–2 个就够），系统自动收敛到更小的内容集合，再让用户看推荐与继续筛选。
- 四维不是四个独立频道，而是同一套“内容标签（Facet）”的不同入口视角：入口不同，但最终落到同一份检索与推荐能力上。

#### 一

  步骤 1：定义四维场景体系与标签规范（先做这一步，后面才不会乱）

  ##### 1. 场景词表（第一版草案）
  > 控制在 10–20 个高频标签，便于后续迭代扩展。括号内是建议同义词或备注，必要时维护父级标签。

  - **人群（12 个）**  
    儿童健康、青春期学生、大学生、职场久坐族、996 人群、夜班工作者、孕产妈妈、备孕家庭、中老年群体、银发族、慢病人群-高血压、慢病人群-糖尿病

  - **症状（16 个）**  
    失眠（入睡困难/睡不着）、浅睡易醒、晨起疲惫、焦虑紧张、情绪低落、头痛头胀、肩颈僵硬、久坐腰痛、乏力倦怠、胃胀消化不良（胃不舒服）、便秘排便困难、食欲不振、口干咽痛、易感冒、皮肤干燥瘙痒、女性经期不适

  - **季节 / 节气（10 个）**  
    春季养生、夏季防暑、秋季润燥、冬季御寒、雨水节气、清明节气、夏至节气、立秋节气、立冬节气、腊八前后

  - **目标（15 个）**  
    减脂塑形、增肌提能、体重管理、控糖控压、睡眠改善、缓解压力情绪、肠胃调理、免疫力提升、体质调理-偏寒、体质调理-偏热、术后康复、产后恢复、学生专注力、护眼放松、气血调养

  ##### 2. 标签使用规则

  - **标签结构**  
    - 每篇内容必须填写 1 个“主标签”（四维之一）+ 2–5 个“辅标签”（可跨维度）。  
    - 主标签优先顺序：症状 > 目标 > 人群 > 季节（若命中多个症状，以文章核心主题为准）。
    - 同一维度可多选辅标签，但最多 3 个，避免泛化。

  - **命名规范**  
    - 统一使用简体中文、无需标点；若为组合标签使用“·”连接（例：上班族·久坐疲劳）。  
    - 子标签统一前缀（例：慢病人群-高血压），便于检索与分组。

  - **同义词与层级策略**  
    - 建“同义词 → 标准标签 ID”映射表，内容运营录入时自动提示标准写法。  
    - 常见同义词示例：  
      - 失眠、睡不着、入睡困难 → 失眠  
      - 胃不舒服、胃胀、消化不良 → 胃胀消化不良  
      - 颈椎酸痛、肩颈僵硬、落枕 → 肩颈僵硬  
    - 标准标签维持平铺结构，必要时通过 `parentId` 建父子关系（如“慢病人群”之下的细分）。

  - **禁用/谨慎标签列表**  
    - 涉及明确诊疗方案或危险病情（如“癌症治疗”“心梗急救”）暂不纳入标签体系，转向“就医提示”内容。  
    - 如需描述传染病等高敏词，必须附带“请遵医嘱/及时就医”提示，并由医疗审核同意。  
    - 定期检查新增标签，若超出健康管理范围需二次确认。

  - **维护流程**  
    - 词表、同义词库由“内容运营 + 医疗审核”共同维护，每周例会评审新增需求。  
    - 任何新增/禁用标签需在《标签变更记录》文档登记：标签名称、维度、用途、审批人、上线日期。  
    - 每月输出一次“标签使用报告”，统计各维度标签覆盖率、未被使用的标签、热门组合场景。

  ##### 3. 产出物

  - 《四维标签词表.xlsx》：四个 Sheet，对应人群/症状/季节/目标，字段包含 `标签ID/标签名称/描述/同义词/父级标签/使用示例/是否启用`。  
  - 《标签规范说明.md》：梳理本文规则，附常见问答（如如何判断主标签、如何处理不确定标签）。  
  - 《标签变更记录.xlsx》：维护变更历史，支持后续审计。  
  - 《敏感标签清单.md》：记录禁用/谨慎标签及处理策略，供内容审核使用。

  #### 二
  
    步骤 2：设计场景入口信息架构与交互流程（降低搜索成本的核心）
  
- 入口形态建议（从轻到重）
  - 轻量：顶部“场景快捷入口”4 个 Tab（人群/症状/季节/目标）+ 热门标签 Chips。
  - 标准：一个“场景选择器”页（类似向导），支持“先选 1 个维度 → 给出推荐内容 → 再补充第 2 个维度精炼结果”。
  - 进阶：根据用户画像/近期行为默认预选（比如冬季默认“冬季”，或根据打卡数据推荐“睡眠改善”）。
- 交互细节（让用户更快到结果）
  - 先给“推荐场景卡片”（如“上班族·久坐疲劳”“冬季·易感冒”“失眠·入睡困难”），点击直接落到结果页。
  - 结果页始终可见当前已选标签（可一键删除），避免用户迷路。
  - 搜索框做“场景内搜索”：已选场景会作为过滤条件，搜索只在该集合内进行。
步骤 3：改造内容模型与检索召回策略（让四维能跑起来）

- 内容最小必备字段
  - title/summary/body
  - tags（四维 Facet + 同义词规范后的标准 ID）
  - evidenceLevel（可选：经验/参考/指南等，用于排序与免责声明策略）
  - updatedAt（用于时效性）
- 召回与排序建议
  - 召回：先按标签过滤（人群/症状/季节/目标）→ 再关键词/语义检索兜底（避免“选错标签就空”）。
  - 排序：标签匹配度（主标签命中优先）+ 时效性 + 质量分（人工/运营可调）+ 个性化信号（有就用，没有不强求）。
  - 空结果策略：自动放宽一档（比如从“人群+症状”放宽到“症状”），并提示“已为你扩大范围”。
步骤 4：内容生产、治理与评估指标（不做治理会越来越乱）

- 生产与治理流程
  - 先挑 50–100 篇高访问内容做“补标签”，跑通闭环，再扩到全量。
  - 每周做一次标签健康检查：Top 场景是否有内容？是否某些标签内容过少/过泛？
- 核心指标（衡量是否真的降低搜索成本）
  - 场景入口使用率（入口点击/UV）
  - 首次命中率（进入知识库后 N 次点击内找到并停留 ≥X 秒）
  - 搜索次数下降（同一会话 search 次数、search→退出率）
  - 0 结果率与放宽后转化率
  - 内容满意度（收藏/点赞/“有用”反馈）
