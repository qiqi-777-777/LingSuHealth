# 节气挂件修复说明

## 问题原因
前端代理配置错误，指向了错误的后端端口。

### 错误配置
```typescript
// vite.config.ts
proxy: {
  '/api': {
    target: 'http://localhost:8080', // ❌ 错误：后端实际运行在 8081
    changeOrigin: true,
  },
}
```

### 正确配置
```typescript
// vite.config.ts
proxy: {
  '/api': {
    target: 'http://localhost:8081', // ✅ 正确：后端运行在 8081
    changeOrigin: true,
  },
}
```

### 1. 修改配置文件

## 技术细节

### 为什么会出现这个问题？
1. 后端配置文件 `application.yml` 中设置端口为 8081
2. 但 Vite 配置文件中代理仍指向 8080
3. 导致前端请求无法到达后端
4. API 调用失败，组件无法获取数据

### 代理的作用
- 开发环境中，前端（5174）和后端（8081）端口不同
- 浏览器的同源策略会阻止跨域请求
- Vite 代理将 `/api` 请求转发到后端
- 这样前端可以像访问同域一样访问后端 API
